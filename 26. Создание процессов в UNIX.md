# 26 Билет
## Теория
**Создание процессов в UNIX-подобных операционных системах.**
В UNIX-подобных системах создание нового процесса осуществляется с помощью системного вызова fork(). Он выполняет две основные задачи:

1. Создание копии текущего процесса (родителя) — получается новый процесс (дочерний), который абсолютно идентичен родителю, кроме уникальных идентификаторов (PID).
2. Возвращение разных значений в родителе и ребенке:
	a.       В родительском процессе возвращается PID (идентификатор) вновь созданного дочернего процесса.
	b.       В дочернем процессе возвращается 0.

Основные функции для работы с процессами:
1. fork():Создает новый процесс (копию текущего), возвращая:

	a.       В родителе: PID дочернего процесса.

	b.       В ребенке: 0.

2. execl(), execv(), execle(), execvp():Семейство функций для запуска новой программы в текущем процессе. Например:
3. execl("/bin/ls", "ls", "-la", NULL) — запустит команду /bin/ls -la.
4. wait():Ожидает завершения дочернего процесса и возвращает его PID.
5. kill():Позволяет отправить сигнал процессу (например, завершить процесс с помощью сигнала SIGTERM).
6. getpid():Возвращает собственный PID текущего процесса.
7. getppid():Возвращает PID родительского процесса.
