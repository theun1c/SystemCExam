## Теория
**Анонимные каналы (в Windows и Linux). Назначение и использование.**

Анонимные каналы — это средство для передачи данных между родственными процессами (родительским и дочерними процессами) в форме однонаправленных каналов связи. Каналы состоят из двух концов: один для записи (writer), другой для чтения (reader). Родительский процесс обычно создает канал, а дочерний процесс наследует его.

Назначение анонимных каналов:
1. Передача данных между родственными процессами.
2. Простота реализации и настройка коммуникаций между родителями и детьми.
3. Ориентация на кратковременные взаимодействия, существующие только в период выполнения программы.

Основные особенности:
1. Однонаправленность: Данные передаются только в одном направлении (от писателя к читателю).
2. Родственные процессы: Каналы создаются для передачи данных между близкими процессами (например, между родительским и дочерним).
3. Кратковременность: Жизнь канала заканчивается с завершением процессов.
