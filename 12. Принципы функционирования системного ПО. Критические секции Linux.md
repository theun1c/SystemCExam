## Теория 

**Принципы функционирования системного программного обеспечения (на основе анализа отличий от других типов ПО)**

**Системное ПО** - это комплекс программных средств, обеспечивающих работу вычислительной системы и взаимодействие между аппаратным обеспечением, пользователем и прикладными программами. Оно служит основой для функционирования всех остальных приложений и управляет ресурсами компьютера.

Основная цель системного ПО — предоставить удобный интерфейс для взаимодействия с компьютером и эффективно управлять его ресурсами. 

Отличия от прикладного ПО:
1. Системное: работает с устройствами, управляет ресурсами.
2. Прикладное: браузер, игры и так далее, не знает про железо

Функции системного ПО:
1. Обеспечение работы устройств: Устанавливает связь между программами и физическим оборудованием (через драйверы устройств).
2. Запуск и остановка программ: Организует загрузку и выгрузку программ, управление памятью и доступом к процессорам.
3. Безопасность и защита: Мониторит активность программ, защищает от несанкционированного доступа, вирусных угроз.
4. Оптимизация ресурсов: Распределяет ресурсы компьютера (процессор, память, хранилища) между различными задачами.
5. Сервисы и утилиты: Поддерживает инструменты для настройки, обслуживания и диагностики компьютера.

Примеры системного ПО:
1. Операционные системы: Windows, macOS, Linux.
2. Драйверы устройств: драйверы видеокарт, звуковых карт, принтеров.
3. Антивирусные программы: защищают систему от вредоносных воздействий.
4. Утилиты: диспетчеры задач, менеджеры загрузки, средства очистки диска.

## Практика 
Создание программы с критической секцией в ОС Linux. Программа должна содержать минимум два потока. Использование критической секции в функции потока должно быть обосновано.

Шаг 1. В файле main.c на ОС Linux написать следующий код.

``` C

int counter = 0;
pthread_mutex_t mutex;

void* pthreadFun(){
	for(int i = 0; i < 10000000; i++){
		pthread_mutex_lock(&mutex);
		counter++;
		pthread_mutex_unlock(&mutex);
	}
	pthread_exit(0);
}

int main(){
	pthread_mutex_init(&mutex, 0);
	
	pthread_t t1, t2, t3;
	pthread_create(&t1, 0, pthreadFun, 0);
	pthread_create(&t2, 0, pthreadFun, 0);
	pthread_create(&t3, 0, pthreadFun, 0);
	
	pthread_join(t1, 0);
	pthread_join(t2, 0);
	pthread_join(t3, 0);
	
	printf("%d\n", counter);
	
	pthread_mutex_destroy(&mutex);
	return 0;
}
```

Шаг 2. Ввести следующие команды в терминале проекта.

``` 
gcc main.c
```
