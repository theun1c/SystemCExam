## Теория 

**Особенности разработки программ с использованием API информационной системы (основные отличия от программ, не использующих API)  
**API (Application Programming Interface) — это программный интерфейс, предоставляющий набор готовых методов и классов для взаимодействия с какой-либо платформой, сервисом или инфраструктурой. Использование API имеет ряд отличий и преимуществ по сравнению с разработкой классических приложений без подключения внешних сервисов или платформ.

Чем отличается разработка с API от классической?
1. Классическая разработка (без API):

	 1.       Самостоятельно реализуете весь функционал, включая работу с файлами, базами данных, сетевые взаимодействия и бизнес-логику.

	 2.       Большой объем ручной работы и необходимость детальной проработки всех деталей.

	 3.       Больше рисков допустить ошибку в низкоуровневой части (SQL-запросы, форматирование данных и т.д.)

2. Разработка с использованием API информационной системы:

	 1.       Получаете доступ к готовым методам и структурам данных, предоставляемым платформой (например, 1С, SAP, Oracle).

	2.       Ваша задача сводится к настройке параметров вызова и обработке результатов API.

	3.       Готовая инфраструктура платформы снимает нагрузку с разработчика и устраняет большую часть технических нюансов.

Основные преимущества использования API:

1.       Быстрота разработки: Готовые методы избавляют вас от рутины, что ускоряет выпуск готового продукта.

2.       Минимизация ошибок: Проблемы решаются на стороне платформы, ваши задачи становятся простыми и понятными.

3.       Расширенные возможности: Внешние API открывают доступ к дополнительным возможностям, которые невозможно было бы реализовать собственными силами (например, определение лиц на фотографиях через API).

## Практика 
Получение названия клавиши с использованием функции GetKeyNameText (Windows HOOK). Название клавиши выводить в MessageBox

Шаг 1. Создать файл main.c с следующим кодом.

``` C
#include <Windows.h>

LRESULT CALLBACK HookProc(int code, WPARAM wParam, LPARAM lParam) {
	if (code >= 0 && wParam == WM_KEYDOWN) {
		PKBDLLHOOKSTRUCT pKeyboard = lParam;
		char* buffer[100];
		if (GetKeyNameTextA(pKeyboard->scanCode << 16, buffer, 100) != 0){
			MessageBoxA(0, buffer, "Title", 0);
		}
	}
	else {
		CallNextHookEx(0, code, wParam, lParam);
	}
}

int main() {
	HHOOK hook = SetWindowsHookExA(WH_KEYBOARD_LL, HookProc, GetModuleHandleW(0), 0);
	if (hook != 0) {
		MSG msg;
		while (GetMessageW(&msg, 0, 0, 0)) {
			TranslateMessage(&msg);
			DispatchMessageW(&msg);
		}
		UnhookWindowsHookEx(hook);
	}
}
```
